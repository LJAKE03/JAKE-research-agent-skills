# Research Agent Operating Rules

对任何包含以下任一特征的任务，必须首先读取并遵循 `00-research-orchestrator/SKILL.md`：

- 需要多个步骤；
- 涉及科研项目、论文、综述、报告或技术路线；
- 涉及多个文件、数据源、模型或工具；
- 结论需要外部证据；
- 用户只给出了较泛的目标；
- 完成质量明显依赖用户补充信息。

强制规则：

1. 总控是唯一默认入口和阶段返回点。
2. 不得同时加载所有 Skill；只加载当前阶段所需 Skill。
3. 每个功能 Skill 必须返回 `shared/STAGE_HANDOFF.template.md` 所定义的交接包。
4. 每个阶段必须由 `06-quality-gate/SKILL.md` 检查，并由总控决定通过、返工、补充信息或停止。
5. 不得用一次回复完成完整科研项目、整篇论文或全部报告，除非任务非常短且用户明确要求。
6. 对可以通过文件、数据库或互联网查明的信息，先自行查找；不要把检索工作转嫁给用户。
7. 对研究目标、范围、偏好、资源约束和风险容忍度等只有用户能决定的问题，主动询问。
8. 维护最新项目状态；不要重复提问或重复执行已经确认完成的工作。

应用版 Codex 的模型分层以 `.codex/config.toml` 和 `.codex/agents/*.toml` 为准：主代理负责战略与最终审核，`research_support` 仅给候选证据，`research_output` 只做已确认内容的机械处理。具体模型可用性和工具规则见 `shared/MODEL_ROUTING.md`。
