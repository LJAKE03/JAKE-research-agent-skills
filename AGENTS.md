# Research Skills Operating Rules

对任何包含以下任一特征的任务，必须首先读取并遵循 `00-research-orchestrator/SKILL.md`：

- 科研项目、论文、综述、报告、方法或技术路线；
- 多个步骤、文件、数据源或工具；
- 需要外部证据或高影响科研判断；
- 目标较泛，或质量依赖用户独有信息。

本仓库只有一条科研流程。用户不选择模型、Agent 或工作模式；总控自动决定是否检索、调用哪个功能 Skill、是否委派、是否分阶段和使用何种质量检查。

强制规则：

1. Sol 总控是唯一默认入口、阶段返回点和最终用户接口。
2. Terra 只接收有界证据任务卡，执行检索、网页查证、文件扫描、提取和证据表整理。
3. Luna 只接收完整锁定写作包，生成文本、语言和格式，不新增事实、引用、公式或科研判断。
4. 研究方向、任务拆解、方法、参数、边界、来源可靠性、证据综合和最终结论由 Sol 负责。
5. 只有真实依赖才创建阶段；简单、低风险任务直接完成。
6. Worker 不得互相转交或递归委派，统一按紧凑交接 Schema 返回 Sol。
7. 模型间不复制完整对话、全部项目历史、全部工具日志或整篇论文原文。
8. 投稿、申报、安全或高成本、关键参数、核心方法和最终科学结论由 Sol 做一次紧凑语义验收，不重新写全文。
9. 只在阶段切换、跨轮恢复或高影响变化时更新项目状态。
10. 对公开可查信息先自行查找；只询问用户独有且会改变路线或交付物的问题。
11. 本地科研代码的跨文件结构、数据流、复现路径或改动影响可按需调用 `07-code-context`；CodeGraph 只是可选外部检索后端，不自动安装或初始化，缺失时回退 `rg` 和定点读取。

Windows PowerShell 读取仓库文本时显式使用 `Get-Content -Encoding UTF8`；不要用默认编码或 `gc`/`cat`/`type` 猜测解码。优先用 `rg`、限定行或摘要读取大型文件；文本修改使用 `apply_patch`。需要由 Windows PowerShell 5.1 执行且含非 ASCII 的 `.ps1` 必须保留 UTF-8 BOM。

应用版 Codex 的模型路由以 `.codex/config.toml`、`.codex/agents/*.toml` 和 `shared/MODEL_ROUTING.json` 为准。主代理使用 Sol；`research_support` 使用 Terra；`research_output` 使用 Luna。
