# Skill 路由示例

## 示例 1：泛化科研项目

用户：

> 帮我做船用氢燃料供给系统电动阀故障预测研究。

路由：

1. `00` 总控识别为复杂项目；
2. `01` 询问故障对象、预测目标、数据来源、验证条件和交付物；
3. `02` 检索相似阀门故障预测工作流、传感器配置和评价方法；
4. `03` 拆成数据方案、故障注入、模型开发、验证、论文输出；
5. 每阶段完成后调用 `06`；
6. 需要综述时调用 `04`；
7. 论文成稿时调用 `05`。

## 示例 2：SCI 小论文

用户：

> 根据现有仿真结果写一篇 SCI 小论文。

路由：

1. `00` 检查是否已有研究问题、数据、图表和目标期刊；
2. `01` 补齐只有用户能决定的信息；
3. `02` 核查相关研究、方法和创新性；
4. `03` 拆成论点—证据图、大纲、方法、结果讨论、引言结论、全文校核；
5. `05` 逐章节写；
6. 每章经 `06` 后再写下一章。

## 示例 3：文献综述

用户：

> 做一篇预测性维护综述。

路由：

1. `00` 确定是否为课程综述、SCI 综述或项目背景；
2. `01` 明确对象、时间范围和深度；
3. `02` 检索综述框架、代表性论文和最新进展；
4. `04` 完成检索策略、文献矩阵、主题综合和缺口；
5. `06` 检查证据覆盖；
6. `05` 转化为正式文章或报告。

## 示例 4：已有报告修改

用户：

> 根据专家意见把项目总结报告全部改好。

路由：

1. `00` 读取报告、意见和格式要求；
2. `01` 只询问意见中无法判断的优先级；
3. `03` 按“结构—证据—创新点—建议—格式”拆成工作包；
4. `05` 分章节修改；
5. `06` 对照每条意见逐条销项；
6. 总控给用户修改清单和最终版本。
